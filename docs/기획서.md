# [기획서] AI 자동화 기반 가상화폐 에어드랍 정보 큐레이션 플랫폼

---

## 1. 프로젝트 개요

**프로젝트 목적**
파편화된 가상화폐 에어드랍 정보를 자동으로 수집하고, AI를 통해 스캠을 필터링 및 규격화하여 사용자에게 안전하고 정제된 정보만을 제공하는 모바일 애플리케이션 구축.

**타겟 유저**
매일 에어드랍 및 바운티 작업에 참여하며, 스캠 정보에 피로도를 느끼는 가상화폐 투자자.

**개발 환경 및 기술 스택**
*   **프론트엔드:** React Native, Expo
*   **백엔드:** Node.js, Express, bcryptjs, jsonwebtoken
*   **데이터베이스:** MongoDB
*   **AI API:** Gemini 1.5 Flash (사용 가능한 모델명 확인 필요)
    > **참고:** `gemini-1.5-flash` 모델이 `404 Not Found` 오류를 발생시킬 경우, 사용자 계정 또는 지역에서 해당 모델이 지원되지 않을 수 있습니다. `backend` 디렉터리에서 `node list_gemini_models.js`를 실행하여 사용 가능한 모델 목록을 확인하고, 그 중 하나를 `backend/src/services/scraper.js` 파일의 `model` 값으로 변경해 주세요. (예: `gemini-pro`, `gemini-1.5-pro` 등)
*   **수집 도구:** Axios, rss-parser

## 2. 시스템 아키텍처 및 파이프라인

전체 프로세스는 사람의 개입 없이 100% 자동화로 동작합니다.

1.  **Data Collector:** Node.js 크롤러가 타겟 채널(RSS 피드)에서 1시간 주기로 Raw Data 수집.
2.  **AI Gatekeeper:** 수집된 텍스트를 Gemini API로 전송.
3.  **Data Processing:** Gemini가 스캠 여부 판별(점수화) 및 JSON 포맷으로 필수 데이터 추출.
4.  **Database Storage:** 신뢰도 점수가 기준치(예: 70점) 이상인 정상 데이터만 MongoDB에 적재.
5.  **Client App:** Expo로 빌드된 모바일 앱에서 DB 데이터를 호출하여 피드(Feed) 형태로 렌더링.

## 3. 기능 요구사항 명세

### 3.1 백엔드 파트 (데이터 파이프라인)

*   **크롤링 스케줄러:** 외부 Cron Job 또는 스크립트 실행기로 주기적 실행. **(구현 완료)**
*   **중복 방지 로직:** 게시글의 고유 URL 또는 메시지 ID를 해시 처리하여 기존 DB와 대조, 중복 수집 방지. **(구현 완료)**
*   **AI 프롬프트 엔지니어링:** Gemini API 호출 시 `responseMimeType: "application/json"`을 강제하여 응답 포맷 고정. **(구현 완료)**
*   **스캠 필터링 기준:** 지갑 개인키 요구, 사전 입금 요구 등을 기반으로 AI가 스캠 및 신뢰도 점수 판별. **(구현 완료)**

### 3.2 프론트엔드 파트 (모바일 앱)

*   **사용자 인증 (회원가입/로그인):** 이메일/비밀번호 기반의 사용자 인증 시스템. **(구현 완료)**
*   **에어드랍 탭 (Airdrop Tab):** 최신순/마감임박순/신뢰도순 정렬 기능이 포함된 무한 스크롤 리스트 뷰. **(구현 완료)**
*   **뉴스 탭 (News Tab):** 일반 코인 뉴스 피드. AI가 에어드랍이 아니라고 분류하고 스캠이 아닌 항목 표시. **(구현 완료)**
*   **캘린더 보기 (Calendar View):** 에어드랍 마감일을 달력 형태로 표시하고, 특정 날짜의 에어드랍 목록 제공. **(구현 완료)**
*   **상세 페이지 (Detail View):** 에어드랍 참여 방법 3줄 요약, 보상 규모, 공식 링크 제공. **(구현 완료)**
*   **푸시 알림 (Push Notification):** 신뢰도 90점 이상 에어드랍 발생 시, 등록된 모든 사용자에게 푸시 알림 발송. **(구현 완료)**
*   **인앱 브라우저 (In-App Browser):** 참여 링크 클릭 시 외부 브라우저 이탈 없이 앱 내부에서 웹 뷰를 띄워 체류 시간 확보. **(구현 완료)**
*   **실시간 시장 가격 데이터:** 토큰 티커가 있는 에어드랍에 대해 실시간 가격 및 24시간 변동률 표시. **(구현 완료)**

## 5. 상세 예외 처리 및 엣지 케이스 방어 로직

서비스의 안정성을 결정짓는 가장 중요한 파트입니다. 각 구간별 발생 가능한 예외 상황과 대처 방안을 명세합니다.

### 5.1 수집기(크롤러) 예외 처리

*   **타겟 웹사이트 DOM 구조 변경:** 데이터 수집 값이 `null`이거나 빈 문자열일 경우, DB 저장을 즉시 중단하고 관리자에게 웹훅(Slack/Discord) 에러 알림 발송.
*   **IP 차단 (Rate Limit):** 대상 서버에서 요청을 거부(HTTP 429 Too Many Requests)할 경우, 지수 백오프(Exponential Backoff) 알고리즘을 적용하여 재시도 간격을 점진적으로 늘림.
*   **텔레그램 채널 폭파 및 비공개 전환:** API 응답 에러 발생 시 해당 소스를 일시적으로 수집 타겟에서 제외(Disable) 처리.

### 5.2 AI (Gemini API) 예외 처리

*   **API 응답 시간 초과 (Timeout):** 요청 후 15초 이상 응답이 없으면 해당 데이터 처리를 큐(Queue)의 맨 뒤로 미루고 다음 데이터 먼저 처리.
*   **JSON 포맷 파괴 (Hallucination):** Gemini가 지정한 JSON 형식을 무시하고 일반 텍스트나 깨진 괄호를 반환할 경우, 정규식(`try-catch` 블록 내 `JSON.parse`)으로 1차 복구 시도. 복구 실패 시 해당 데이터 폐기(Drop).
*   **필수 데이터 누락:** 신뢰도 점수는 90점이나 `official_link` 값이 빈 문자열로 반환될 경우, 앱 노출 시 심각한 결함이 되므로 DB 적재 프로세스에서 강제 필터링(Drop).

### 5.3 서버 및 DB 예외 처리

*   **데이터 중복 적재 충돌:** 동일한 에어드랍 정보가 다른 소스에서 수집될 경우, 고유 인덱스(Unique Index) 위반 에러가 발생. 이 경우 `upsert` 로직으로 전환하여 기존 데이터의 '출처(Source)' 정보만 배열에 추가.
*   **트래픽 스파이크:** 푸시 알림 발송 직후 다수의 유저가 동시 접속하여 DB 커넥션 풀(Connection Pool)이 고갈될 경우, 리드 레플리카(Read Replica)를 활용하거나 Redis 캐싱 계층을 두어 부하 분산.

### 5.4 클라이언트(모바일 앱) 예외 처리

*   **네트워크 단절 (Offline):** 유저의 데이터 통신이 끊겼을 때 빈 화면을 노출하지 않도록, 이전에 불러온 1 페이지 분량의 데이터를 로컬 스토리지에 캐싱하여 렌더링.
*   **무한 로딩 버그 (Spinner Lock):** 백엔드 응답이 지연될 때 로딩 스피너가 영원히 도는 것을 방지하기 위해 10초 타임아웃 설정. 10초 경과 시 "정보를 불러오는 중 문제가 발생했습니다. 다시 시도해 주세요" 안내 문구 및 새로고침 버튼 렌더링.
*   **외부 링크 접속 불가 (HTTP 404):** 유저가 '참여하기' 버튼을 눌렀으나 대상 사이트가 닫힌 경우, 앱 내부에서 에러를 핸들링하여 "현재 해당 프로젝트 웹사이트에 접근할 수 없습니다"라는 토스트 알림 노출.

## 6. 수익화(광고) 배치 전략

안드로이드 기반 구글 애드몹(AdMob) 연동 기준입니다.

*   **하단 배너 광고 (Banner):** 메인 피드와 상세 페이지 하단에 항상 고정 노출하여 기본 노출수(Impression) 보장. **(구현 완료)**
*   **피드 네이티브 광고 (Native):** 리스트 항목 7개당 1개 꼴로 앱의 UI와 완벽하게 일치하는 디자인의 네이티브 광고 삽입. **(구현 완료)**
*   **보상형 동영상 (Rewarded):** '신뢰도 95점 이상 초고수익 예상 에어드랍' 정보 열람 시, 15초 광고 시청 후 잠금 해제하는 방식으로 객단가(eCPM) 극대화.
