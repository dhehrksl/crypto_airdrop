## Context

뉴스 파이프라인(`backend/src/services/scraper.js`)은 27개 RSS 출처와 Snapshot 거버넌스 데이터를 수집해 두 경로로 분기한다.

- **휴리스틱 경로** (`saveHeuristic`): 키워드 점수가 `AI_THRESHOLD`(5) 미만인 항목. AI 호출 없이 `News`에 직접 저장. 비용(Gemini 무료 tier 1,500 RPD)을 아끼는 경로이며, 에어드랍 키워드가 없는 일반 뉴스 대부분이 여기로 온다.
- **AI 경로** (`saveAiResult`): 점수가 `AI_THRESHOLD` 이상인 항목을 배치로 Gemini에 보내 분류·요약. 프롬프트에 `LEGAL SAFETY RULES`가 명시돼 있어 원문 재출판을 금지하고 한 문장 한국어 요약만 만든다.

문제는 휴리스틱 경로가 `item.contentSnippet`(출처 원문 발췌)을 240자로 잘라 `description`에 그대로 저장한다는 점이다. AI 안전 계층을 우회하므로, 화면에는 가공되지 않은 영어 원문 발췌가 노출된다. 한편 `NewsScreen` 부제는 "AI로 번역해 제공"이라 표시하고, `docs/terms-of-service.md` 제3조는 "헤드라인·요약·메타데이터만 가공"이라 진술한다 — 셋 다 휴리스틱 경로의 실제 동작과 불일치한다.

추가로 `docs/terms-of-service.md`는 앱 내 `TERMS_OF_SERVICE_KO`보다 오래된 버전으로, 제거된 `airdrops.io` 출처를 여전히 명시한다. Play Console 등록에는 호스팅된 정책 URL이 필요한데 `PRIVACY_POLICY_URL`/`TERMS_URL`이 비어 있고, 두 마크다운 모두 시행일자·연락처 placeholder가 미작성 상태다.

## Goals / Non-Goals

**Goals:**

- 휴리스틱 경로가 출처 원문 본문을 복제하지 않게 한다 — 헤드라인 + 링크 + 정형 안내 문구만 저장.
- 앱 UI 표시 문구를 실제 동작과 일치시킨다.
- 호스팅용 법적 문서를 앱 내 버전과 동기화하고, 코드로 채울 수 있는 placeholder를 모두 작성한다.
- RSS 출처 라이선스 리스크를 출시 체크리스트에 명확히 정리한다.

**Non-Goals:**

- AI 호출량을 늘려 모든 뉴스를 요약하는 것 — 무료 tier 한도와 0원 운영 원칙에 어긋난다.
- 변호사 검토, 사업자 상호·관할법원 확정 — 사용자 결정 사항.
- 개별 RSS 출처와의 라이선스 협상·동의 — 사용자 결정 사항(체크리스트로만 안내).
- 뉴스 기능 자체의 제거나 재설계.

## Decisions

### 결정 1: 휴리스틱 뉴스는 본문 발췌 대신 정형 안내 문구를 저장한다

`saveHeuristic()`에서 `snippet`을 잘라 만들던 `description`을 제거하고, 기존에 fallback으로 쓰던 정형 문구("원문 링크에서 상세 내용을 확인하세요.")를 항상 `description`으로 사용한다. `title`(출처 헤드라인)과 `official_link`(원문 링크)는 그대로 유지한다.

- **대안 A — 모든 뉴스를 AI로 보냄**: 법적으로 가장 깨끗하지만 매시 cron에서 무료 tier RPD를 초과할 위험. 0원 운영 원칙 위배. 기각.
- **대안 B — 휴리스틱 항목을 아예 `News`에 저장하지 않음**: 뉴스 탭이 에어드랍 관련 항목만 남아 빈약해진다. 헤드라인+링크 수집은 일반적인 뉴스 aggregator 관행으로 위험이 낮으므로 과한 조치. 기각.
- **채택 — 헤드라인 + 링크 + 정형 안내**: Google News 식 헤드라인 aggregation에 해당. 원문 발췌 복제만 제거하면 저작권 리스크가 크게 낮아진다. 코드 변경이 최소(한 함수)이고 비용 증가 0.

영어 헤드라인을 그대로 두는 것은 의도된 선택이다 — 헤드라인은 사실 전달용 짧은 문구로 저작물성이 약하고, 출처 표기와 원문 링크가 함께 제공된다. 헤드라인 번역까지 하려면 AI 호출이 필요해 비용 원칙과 충돌한다.

### 결정 2: UI 부제를 실제 동작에 맞춘다

`NewsScreen`의 "제3자 출처의 암호화폐 뉴스를 AI로 번역해 제공합니다"를, 헤드라인 수집이 기본이고 일부만 AI 요약된다는 사실을 반영한 문구로 교체한다(예: "제3자 출처의 암호화폐 뉴스 헤드라인을 모아 제공합니다. 상세 내용은 원문에서 확인하세요."). "AI 번역"을 보편적 사실처럼 단정하지 않는다.

### 결정 3: `docs/`의 법적 문서를 앱 내 `policies.js` 버전을 정본으로 동기화한다

`TERMS_OF_SERVICE_KO`/`PRIVACY_POLICY_KO`는 이미 갱신돼 있고 앱이 실제로 보여주는 본문이므로 이를 정본으로 삼는다. `docs/terms-of-service.md`를 여기에 맞춰 `airdrops.io` 명시를 제거하고 출처 목록을 일치시키며, 제3조 저작권 문구를 "헤드라인과 짧은 문맥 요약, 원문 링크만 제공"으로 정정한다. `docs/privacy-policy.md`는 내용이 대체로 일치하므로 placeholder 작성과 HTTPS 단정형 정리만 한다.

### 결정 4: placeholder는 코드로 확정 가능한 것만 채우고 나머지는 명시적으로 남긴다

- **채울 수 있음**: 시행일자(출시 예정 기준 날짜), 문의/개인정보 보호책임자 연락처(운영자 이메일 `kimminsu4906@gmail.com`).
- **남겨둠**: 사업자 상호, 관할법원, 보호책임자 실명 등 — `[변호사 검토 후 기입]`처럼 명확한 표식으로 남기고 `launch-checklist.md`에 연결한다.

`PRIVACY_POLICY_URL`/`TERMS_URL`은 외부 호스팅이 선행돼야 하므로 코드에서 값을 채울 수 없다. 체크리스트 항목으로만 유지한다.

## Risks / Trade-offs

- **기존 `News` 문서에 남아 있는 원문 발췌** → `News`는 3일 retention(`pruneOldNews`)으로 자동 삭제되므로 배포 후 최대 3일이면 자연 교체된다. 즉시 정리가 필요하면 일회성 스크립트로 휴리스틱 저장분의 `description`을 정형 문구로 덮어쓸 수 있으나, retention만으로 충분하다고 판단한다.
- **뉴스 탭의 정보량 감소** → 휴리스틱 항목 카드가 헤드라인만 보여줘 밋밋해진다. 다만 원문 보기 버튼이 있고, 본문 발췌 복제 리스크 제거가 우선이다. AI 경로 항목은 여전히 한국어 요약을 가지므로 탭 전체가 비지는 않는다.
- **호스팅 정책 URL 부재** → 코드 범위 밖. 체크리스트에 블로커로 명확히 표기해 출시 전 사용자가 처리하도록 한다.
- **영어 헤드라인 잔존** → 한국어 사용자 경험이 다소 떨어지나, 헤드라인은 저작물성이 약하고 비용 원칙상 수용 가능한 트레이드오프.

## Migration Plan

1. `scraper.js`·`NewsScreen.js` 코드 변경 배포 — 이후 신규 수집분부터 정형 `description` 적용.
2. 기존 데이터는 3일 retention으로 자연 교체(별도 마이그레이션 스크립트 불필요).
3. 법적 문서 변경은 문서 파일 수정만으로 완료. 외부 호스팅·URL 등록은 launch-checklist 항목으로 사용자에게 이관.
4. 롤백: 코드 변경은 단일 함수 수준이라 git revert로 즉시 복귀 가능. 데이터 손실 없음.

## Open Questions
