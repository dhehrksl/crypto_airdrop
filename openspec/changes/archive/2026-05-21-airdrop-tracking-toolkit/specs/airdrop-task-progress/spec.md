## ADDED Requirements

### Requirement: 사용자는 에어드랍 참여 단계를 개별 체크·해제할 수 있다

인증된 사용자는 특정 에어드랍의 참여 단계(`Airdrop.tasks` 배열의 각 항목)를 개별적으로 완료 처리하거나 완료 해제할 수 있어야 한다(SHALL). 단계 완료 상태는 사용자별·에어드랍별로 저장되며, 다른 사용자의 진행 상태에 영향을 주지 않는다(MUST NOT).

단계는 `tasks` 배열의 인덱스로 식별한다. 유효 범위를 벗어난 인덱스에 대한 체크 요청은 거부되어야 한다(MUST).

#### Scenario: 참여 단계 하나를 완료 처리

- **WHEN** 인증된 사용자가 에어드랍의 유효한 단계 인덱스를 완료로 표시한다
- **THEN** 해당 단계가 그 사용자의 완료 단계 목록에 포함되고, 갱신된 진행 상태를 반환한다

#### Scenario: 완료한 단계를 다시 해제

- **WHEN** 사용자가 이미 완료 처리한 단계를 해제 요청한다
- **THEN** 해당 단계가 완료 목록에서 빠지고, 갱신된 진행 상태를 반환한다

#### Scenario: 범위를 벗어난 단계 인덱스

- **WHEN** 사용자가 `tasks` 길이를 벗어난 인덱스를 완료로 표시 요청한다
- **THEN** 시스템은 `400 Bad Request`를 반환하고 어떤 진행 상태도 저장하지 않는다

### Requirement: 진행률은 완료 단계 수 기준으로 산출된다

특정 사용자·에어드랍의 진행률은 `완료한 단계 수 / 전체 단계 수`로 산출되어야 한다(MUST). 진행 상태 조회 시 완료 단계 인덱스 목록과 전체 단계 수가 함께 제공되어 클라이언트가 진행률을 표시할 수 있어야 한다(SHALL).

#### Scenario: 일부 단계만 완료한 상태 조회

- **WHEN** 5단계 중 3단계를 완료한 사용자가 해당 에어드랍의 진행 상태를 조회한다
- **THEN** 완료 단계 인덱스 목록과 전체 단계 수(5)가 반환되어 3/5 진행률을 산출할 수 있다

#### Scenario: 참여 단계가 없는 에어드랍

- **WHEN** `tasks` 배열이 비어 있거나 없는 에어드랍의 진행 상태를 조회한다
- **THEN** 시스템은 전체 단계 수 0으로 응답하며, 진행률 산출 대상이 아님을 클라이언트가 알 수 있다

### Requirement: 단계 진행 상태는 인증된 사용자 본인 것만 다룬다

단계 체크·해제·조회는 모두 인증을 요구하며(MUST), 요청자는 자신의 진행 상태만 변경·조회할 수 있어야 한다(MUST). 인증되지 않은 요청은 `401`로 거부되어야 한다.

#### Scenario: 인증 없이 단계 체크 시도

- **WHEN** 인증 토큰 없이 단계 완료 요청을 보낸다
- **THEN** 시스템은 `401 Unauthorized`를 반환한다
