## Requirements
### Requirement: 사용자는 에어드랍을 관심 목록에 추가·제거할 수 있다

인증된 사용자는 특정 에어드랍을 자신의 관심 목록(워치리스트)에 추가하거나 제거할 수 있어야 한다(SHALL). 관심 목록 상태는 `참여 완료` 상태와 독립적으로 관리되며, 한 에어드랍에 대해 두 상태를 동시에 가질 수 있다.

추가/제거 동작은 멱등(idempotent)해야 한다(MUST) — 이미 추가된 항목을 다시 추가하거나, 없는 항목을 제거해도 오류 없이 현재 상태를 반환한다.

#### Scenario: 관심 목록에 에어드랍 추가

- **WHEN** 인증된 사용자가 존재하는 에어드랍 ID로 관심 추가를 요청한다
- **THEN** 해당 에어드랍이 사용자의 관심 목록에 포함되고, 성공 응답을 반환한다

#### Scenario: 이미 관심 등록된 항목을 다시 추가

- **WHEN** 사용자가 이미 관심 목록에 있는 에어드랍을 다시 추가 요청한다
- **THEN** 중복 저장 없이 성공 응답을 반환하며 목록 상태는 변하지 않는다

#### Scenario: 관심 목록에서 제거

- **WHEN** 사용자가 관심 목록에 있는 에어드랍의 제거를 요청한다
- **THEN** 해당 항목이 관심 목록에서 빠지고 성공 응답을 반환한다

#### Scenario: 존재하지 않는 에어드랍 추가 시도

- **WHEN** 사용자가 존재하지 않는 에어드랍 ID로 관심 추가를 요청한다
- **THEN** 시스템은 `404 Not Found`를 반환하고 관심 목록에 아무것도 추가하지 않는다

### Requirement: 관심 목록은 사용자별로 격리되어 조회된다

사용자는 자신의 관심 목록만 조회할 수 있어야 한다(MUST). 인증되지 않은 요청은 거부되어야 하며(MUST), 한 사용자의 관심 목록 조회 결과에 다른 사용자의 관심 항목이 포함되어서는 안 된다(MUST NOT).

#### Scenario: 본인의 관심 목록 조회

- **WHEN** 인증된 사용자가 자신의 관심 목록 조회를 요청한다
- **THEN** 해당 사용자가 추가한 에어드랍 항목만 반환된다

#### Scenario: 인증 없이 관심 목록 조회 시도

- **WHEN** 인증 토큰 없이 관심 목록 조회를 요청한다
- **THEN** 시스템은 `401 Unauthorized`를 반환한다
